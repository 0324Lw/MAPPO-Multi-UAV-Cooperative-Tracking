# 🚀 MAPPO-Multi-UAV-Cooperative-Tracking

> 基于 MAPPO (Multi-Agent Proximal Policy Optimization) 深度强化学习的多无人机协同目标跟踪与动态避障算法。
> 本项目构建了一个高度自定义的多智能体 `gymnasium` 强化学习环境，支持视距 (FOV) 共享、编队包抄协同与高密度静态避障，内置了完整的集中式训练与分布式执行 (CTDE) 管线以及高质量的渲染系统。

## ✨ 核心特性 (Features)

* 🌍 **定制化多智能体微观环境**：构建了 100m x 100m 的二维连续飞行空间，支持多无人机的线速度与角速度连续解算，内置严谨的“防出生点陷阱”鲁棒初始化机制。
* 🛸 **复杂协同与视距追踪**：环境内置随机生成的密集静态障碍物，目标点采用平滑随机游走与被动势场避障策略。无人机受限于扇形视距 (FOV) 与最大探测半径，需通过信息共享实现“抗遮挡”的持续追踪。
* 🧠 **MAPPO 强化学习底座**：采用 **CTDE (集中式训练，分布式执行)** 架构与参数共享 (Parameter Sharing) 机制。Critic 网络拥有统揽全局的 42 维上帝视角，Actor 网络依赖 21 维局部感知。集成了正交初始化、广义优势估计 (GAE)、动作标准差线性衰减等核心机制。
* 📊 **状态空间全归一化**：对无人机获取的相对距离、相对偏航角、速度信息、传感器裕量等状态向量进行严格的 $[-1.0, 1.0]$ 归一化处理，杜绝多尺度输入导致的梯度爆炸，极大提升网络收敛速度。
* 📈 **工业级渲染与数据分析**：内置完备的 `DataLogger` 记录各项成功率与跟踪耗时，并提供基于 Matplotlib 的高级 GIF 渲染系统（含莫兰迪配色、视场角可视化与动态历史轨迹尾迹），直观展现策略的演进。

## 🧠 强化学习环境设计 (Environment Design)

本项目针对多机协同跟踪与避障的耦合难题进行了降维与解耦。为了让无人机学会在复杂环境中进行“包抄”与“互补”，核心的状态空间、动作空间与奖励函数设计如下：

### 1. 状态空间 (Observation Space)

智能体的局部观测状态被设计为 **21 维连续特征向量 (Continuous 21D)**。为了适配 MAPPO 算法，不仅包含自身与目标状态，还显式融入了队友的协同信息：

| 维度索引 | 物理含义 (Description) | 数据范围 / 说明 |
| :---: | :--- | :--- |
| `0:4` | **目标相对状态**：归一化相对距离、相对偏航角、距离裕量 (距离探测边缘有多远) 与角度裕量。 | 基于信息共享机制，队友视野内的目标也可被感知。丢失时使用 `[-1.0]` 占位补齐。 |
| `4:6` | **视线状态位 (LoS Flags)**：自身是否看见目标、队友是否看见目标。 | 用于网络判断当前目标信息的可靠度。 |
| `6:10` | **队友协同状态**：队友的归一化相对距离、相对角度、线速度与角速度。 | 防撞与包抄战术形成的核心依据。 |
| `10:12` | **自身核心状态**：自身当前的线速度与角速度。 | 约束动力学惯性。 |
| `12:21` | **局部障碍感知**：距离最近的 **3 个障碍物**的特征（距离、角度、类型）。 | 若视野内障碍物不足 3 个，自动使用安全特征进行 Padding 补齐。 |

*注：Critic 网络的输入为所有智能体局部观测的拼接融合（Global State，42 维）。*

### 2. 动作空间 (Action Space)

采用 **2 维连续动作空间 (Continuous 2D)**，底层解算基于经典的差分驱动运动学模型：

* **$a_0$ (线速度控制)**：归一化输出 $[-1.0, 1.0]$，环境内映射至无人机的实际线速度 $[0, V_{max}]$。
* **$a_1$ (角速度控制)**：归一化输出 $[-1.0, 1.0]$，环境内映射至无人机的实际角速度 $[-\Omega_{max}, \Omega_{max}]$。

### 3. 奖励函数设计 (Reward Function)

本环境的奖励函数专为“协同保持”而生，结合稠密奖励与事件惩罚：

$$R_{step} = r_{track} + r_{standoff} + r_{dispersion} + r_{smooth} + r_{danger} + r_{lose}$$

* 🟢 **视场跟踪基础分 ($r_{track}$)**：只要目标在有效探测扇形内，基于角度居中程度给予线性正向奖励。
* 🟢 **最佳距离保持 ($r_{standoff}$)**：跟踪不是撞击。引导无人机停留在探测半径的黄金区间 (40% ~ 80%)，过近或过远均会导致奖励衰减。
* 🟢 **协同包抄激励 ($r_{dispersion}$)**：**多机协同的灵魂**。当两架飞机同时锁定目标时，计算两机相对于目标点的夹角余弦值，鼓励拉开阵型（接近 90°），防止同向跟飞导致的视线盲区。
* 🔴 **动作平滑惩罚 ($r_{smooth}$)**：对相邻步动作变化率施加微小二次方惩罚，消除高频抖动。
* 🔴 **危险区软惩罚 ($r_{danger}$)**：当距离障碍物小于 1.5m 时触发随距离递增的二次方斥力。
* 🔴 **目标丢失惩罚 ($r_{lose}$)**：单步的微弱惩罚，提供持续寻找目标的内在动力。

**🏆 终局截断与稀疏奖励防崩溃设计：**
为了防止多智能体环境中的梯度方差过大：
1. **常规步幅截断**：将所有连续动作项的组合奖励严格限幅 (Clip) 在 **$[-2.0, 2.0]$** 之间。
2. **终局事件直通**：飞机互撞、撞击障碍物、彻底丢失目标将触发 $[-10.0, -20.0]$ 的一次性极刑并结束回合；坚持至最大步数视为胜利，给予 $[+50.0]$ 奖励。此类稀疏信号绕过截断器，提供最清晰的价值导向。

## 🛠️ 环境依赖 (Requirements)

本项目基于 Python 3.10 开发，推荐使用国内清华源极速配置环境：

核心依赖库：
* `torch` (PyTorch >= 2.0)
* `gymnasium`
* `numpy`
* `pandas`
* `matplotlib`
* `scipy`

**快速安装指南 (Conda + Pip)：**
```bash
# 1. 创建并激活虚拟环境
conda create -n uav_mappo python=3.10 -y
conda activate uav_mappo

# 2. 安装科学计算与强化学习基石
conda install numpy pandas matplotlib scipy gymnasium -c conda-forge -y

# 3. 安装 PyTorch (以 CUDA 12.4 为例，推荐使用 pip + 清华源保证速度)
pip install torch torchvision torchaudio --index-url [https://download.pytorch.org/whl/cu124](https://download.pytorch.org/whl/cu124)
```
